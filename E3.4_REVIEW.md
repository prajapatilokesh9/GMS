## 2. Root-Cause Analysis: 403 Forbidden on Commission Routes

### Issue

During integration test verification, all commission endpoints returned **403 Forbidden** with `Insufficient permissions` instead of 201/200.

### Investigation

| Check | Before Fix | After Fix |
|-------|:----------:|:---------:|
| `commission:read` in `permissions` table | ❌ Not found | ✅ Found |
| `commission:manage` in `permissions` table | ❌ Not found | ✅ Found |
| `super_admin` role exists | ✅ Exists | ✅ Exists |
| `super_admin` has `commission:read` assigned | ❌ Missing (0 of 21 perms) | ✅ Present (42 perms) |
| `super_admin` has `commission:manage` assigned | ❌ Missing (0 of 21 perms) | ✅ Present (42 perms) |
| Admin user has `super_admin` role | ✅ Has role (tenant1, null) | ✅ Has role (tenant1-scoped) |
| `loadPermissions` returns commission perms | ❌ Empty | ✅ Contains both |

### Root Cause

The seed file (`apps/backend/src/database/seed.ts`) defined `commission:read` and `commission:manage` permissions at lines 96-97, and the `super_admin` role assignment loop at lines 120-127 assigned all existing permissions. However, the seed had **not been re-run** after these lines were added. The database contained only 21 permissions from the original seed (gym, user, role, audit, settings, tenant categories), with none from the `personal_training` category.

The `commission` routes use `authenticate` + `loadPermissions` + `authorize('commission:*')` middleware. Since `loadPermissions` called `roleService.getUserPermissions()` which returned an empty permission array (the old `super_admin` role had no `commission:*` records in `role_permissions`), the `authorize` middleware correctly returned 403.

### Remediation

1. Ran `npm run db:seed` to upsert the 21 missing permissions (including `commission:read`, `commission:manage`) and assign all 42 permissions to the `super_admin` role
2. Added `beforeAll` hook in the multi-tenancy test to assign `commission:*` permissions to the pilot-tenant's `gym_owner` role (the seed only grants a subset of perms to pilot roles)

### Verification After Remediation

```javascript
// Admin user now has:
// - 2 userRoles (old null-tenant super_admin + new tenant1-scoped super_admin)
// - 42 unique permissions including commission:read ✓ and commission:manage ✓
// - loadPermissions simulation returns both commission permissions
```

## 3. Payout Status Lifecycle

### Allowed Transitions

```
pending ──→ approved ──→ paid
```

- `pending → approved`: POST `/payouts/:id/approve`
- `approved → paid`: POST `/payouts/:id/mark-paid`

### Rejected Transitions

| Transition | Result |
|-----------|--------|
| `pending → paid` (skip approve) | 409 Conflict |
| `paid → approved` | 409 Conflict |
| `paid → paid` | 409 Conflict |
| Generate payout for non-completed session | 409 Conflict |
| Duplicate payout generation for same session | 409 Conflict |

### Commission Calculation

- **Rule precedence:** Trainer-specific rules beat gym-wide rules
- **Within same scope:** Most recent `createdAt` wins
- **Types:** `fixed` (flat amount) and `percentage` (of gross amount)
- **Fallback:** Zero if no active rule found at generation time

## 5. Root-Cause Analysis: 403 Forbidden on Commission Routes

### Issue

During integration test verification, all commission endpoints returned **403 Forbidden** with `Insufficient permissions` instead of 201/200.

### Investigation

The `commission` routes use `authenticate` + `loadPermissions` + `authorize('commission:*')` middleware. `loadPermissions` queries the database via `roleService.getUserPermissions(userId, tenantId)` which collects permission names from all user roles.

The `seed.ts` file defined `commission:read` and `commission:manage` but the seed was **never re-run** after these lines were added. The database contained only the original 21 permissions (gym, user, role, audit, settings, tenant — none from `personal_training` category). The `super_admin` role had zero `commission:*` records in `role_permissions`.

| Check | Before Fix | After Fix |
|-------|:----------:|:---------:|
| `commission:read` in `permissions` table | ❌ Not found (0/42) | ✅ Found (42/42) |
| `commission:manage` in `permissions` table | ❌ Not found (0/42) | ✅ Found (42/42) |
| `super_admin` has `commission:read` assigned | ❌ Missing (0 of 21 role perms) | ✅ Present (42 role perms) |
| `super_admin` has `commission:manage` assigned | ❌ Missing | ✅ Present |
| Admin `getUserPermissions` includes commission | ❌ Empty array | ✅ Contains both |

### Remediation

1. Ran `npm run db:seed` — upserted 21 missing permissions, assigned all 42 to `super_admin`
2. Added `beforeAll` in multi-tenancy test to grant `commission:*` to pilot-tenant's `gym_owner` role

### Verification After Remediation

```javascript

// Has commission:read? true ✅
// Has commission:manage? true ✅
// GET /api/v1/commissions/rules → 200 ✅
```
